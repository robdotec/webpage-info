Sample Article - Benchmarking HTML Parsing

Published on November 24, 2025 by John Doe

Introduction

This is a sample article designed to test the performance of HTML parsing and metadata extraction. It includes various common elements found on modern web pages, including navigation, articles, sidebars, and footers.

The webpage-info library extracts metadata from HTML documents, including titles, descriptions, OpenGraph data, Schema.org structured data, and links. This benchmark measures how efficiently these operations are performed.

Key Features

  • Fast HTML parsing using the scraper library
  • OpenGraph metadata extraction for social sharing
  • Schema.org JSON-LD parsing for structured data
  • Link extraction with URL resolution
  • Text content extraction excluding scripts and styles

Performance Considerations

When parsing large HTML documents, several factors affect performance:

  1. Document size and complexity
  2. Number of elements to traverse
  3. CSS selector matching efficiency
  4. Memory allocation patterns

The library uses cached CSS selectors via OnceLock to avoid repeated parsing, and employs HashSet for O(1) lookups when filtering excluded elements.

Example Usage

Here's how to use the library:

use webpage_info::WebpageInfo;

#[tokio::main]
async fn main() {
    let info = WebpageInfo::fetch("https://example.com").await.unwrap();
    println!("Title: {:?}", info.html.title);
}
